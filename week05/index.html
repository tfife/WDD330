Week 05 - Debugging

Team Activity

Notes

  • Errors are of the following types:
    • System Error
    • Programmer Error
    • User Error
  • Reference Errors occur when something is called (or "referred to") when it doesn't exist where it's expected
  • Stack Trace shows where an error occurred
  • Warnings happen when there's a problem in the code but it doesn't cause the program to fail
  • Strict mode produces more exceptions and warnings
  • Lint programs find non-error sloppy code
  • alert(something) can be used as breakpoints
  • Use console to show info
  • Create JavaScript errors
  • Throw exceptions
  • Use try, catch, and finally
  • Write tests
  • Test Driven Development (TDD)
    • Write tests (that initially fail)
    • Write code to pass the tests
    • Refactor the code
    • Test refactored code
    • Write more tests for new features
  • Jest
